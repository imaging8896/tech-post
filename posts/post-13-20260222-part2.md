# (第2篇/共2篇) AI時代的軟體開發規範：「繁瑣的手寫文件與註解」的消失與「AI協作規範」的新興

## 「繁瑣的手寫文件與註解」的消失
以前，開發者被要求對每個 Function 都要寫詳細的 JSDoc 或註解。然而，現在的AI非常擅長解釋程式碼。比起寫死的文件，現在更強調程式碼的自我解釋性 (Self-documenting)。如果程式碼看不懂，我們會請AI重構到看懂為止，而不是寫一大堆註解來補救。

## 新興的「AI協作規範」
隨著AI和人類開發者的協作越來越密切，「AI協作規範」成為了一個全新的領域。團隊現在需要規範Prompt追蹤、AI生成標記以及安全性掃描等等事項。例如，我們需要追蹤複雜邏輯是否要在註解中留下當時使用的 Prompt，以及哪些區塊是 AI 生成且未經大幅修改的。此外，規範中必須強制要求AI生成的 Code 必須經過特定安全性工具（如 Snyk, SonarQube）的自動化掃描。

## 結語
在AI時代，軟體開發規範正在經歷一場革命。我們從「怎麼寫」轉向「怎麼結構化」，從「繁瑣的手寫文件與註解」轉向「程式碼的自我解釋性」，並且在「AI協作規範」上開闢了新的領域。我們期待這些新的規範能提供更安全、更高效、更清晰的軟體開發環境。
