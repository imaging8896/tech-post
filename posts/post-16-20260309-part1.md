# AI 時代的資料爬蟲：從手動到自動化

(第 1 篇/共 2 篇)

**引言**

在數位資訊爆炸的時代，如何有效地獲取、整理及使用資料成為了極其重要的議題。從網頁上抓取資料（Web Scraping）是一種常見的技術，但隨著網頁結構的複雜化，這項工作變得越來越困難。然而，有了 AI 的幫助，我們能否將這項工作自動化，讓資料爬蟲成為一種更有效率的資料收集方式呢？

**資料爬蟲的挑戰**

傳統的資料爬蟲方式需要透過手動瀏覽網站，找到需要的資料，然後透過瀏覽器的開發工具來尋找可能的 API 或分析網頁的 HTML 結構。這種方式需要花費大量的時間和精力，且需要對網頁結構、API 使用以及程式語言有足夠的了解。此外，由於網頁結構和 API 經常會變動，這種方式在維護上也非常耗時。

**AI 的幫助**

在 AI 的幫助下，這些困難可以得到有效的解決。首先，我們可以問 AI 可能的資料來源，AI 可能就會給出可用的 API 或者網頁。如果 AI 沒有給出有效的來源，我們可以將網址餵給 Github Copilot，Copilot 會嘗試讀取網頁找尋 API，如果無法找到 API，它會讀取 HTML，如果 HTML 也無法讀取，它會讀取 JavaScript 並嘗試執行，從而找到隱藏在 JavaScript 內的 API。
