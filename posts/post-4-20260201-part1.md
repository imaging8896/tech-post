# (第一篇/共兩篇) 未來的程式開發：用 Github Copilot 打造服務的新方法

## 引言

想像一下，你不需要熟悉某個 IDE，不需要手動撰寫程式碼，甚至不需要處理 git commit 和 push，你只需要把需求定義好，透過 Github Copilot，你的服務就能被創建完成。這篇文章會帶領你瞭解如何用 Github Copilot 進行這樣的服務創建。

## 技術問題

隨著 AI 技術的進步，程式碼的撰寫方式也在轉變。Github Copilot 是一種透過 AI 來協助開發者撰寫程式碼的工具，它能夠根據開發者的需求，生成符合需求的程式碼。但是，如何有效利用 Github Copilot 來打造服務，可能對很多開發者來說還是一個新的課題。

## 解決方案

首先，你需要擁有一個 Github 帳號，並且訂閱 Github Copilot。接著，依照以下步驟操作：

1. 登入 Github 首頁後，選擇 **Repositories**，然後點選 **New**。
2. 填入你的專案名稱，最好是能夠符合你想要建立的服務的名稱。
3. 在 **Jumpstart your project with Copilot (optional)** 的 **Prompt** 欄位中，詳細描述你想要的服務的功能。例如，你可以填寫『我希望能夠把我學習到的知識做成文章發佈到 Linkedin』。
