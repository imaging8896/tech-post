# (第1篇/共2篇) AI時代的軟體開發規範：從「怎麼寫」轉向「怎麼結構化」與「AI協作規範」

隨著AI技術的迅速發展，軟體開發規範也在不斷演變。本文將探討在AI時代，哪些軟體開發規範正面臨改變或者已經不再必要。

## 從「怎麼寫」轉向「怎麼結構化」
在過去，軟體開發規範的重點在於寫法（Syntax sugar），但隨著AI的崛起，現在的重點已經轉向模組化與介面設計。此外，規範現在更強調 Type Safety（型別安全）。因為AI在處理強型別（如TypeScript, Rust, Go）時，產出的程式碼準確度遠高於弱型別。設定嚴格的型別規範，是為了「約束 AI」不要亂跑邏輯。

例如，以下是一個 TypeScript 的函數範例：

```typescript
function greet(name: string): string {
    return "Hello, " + name;
}
```

這個函數的型別既明確又安全：它接受一個字符串類型的 `name`，並返回一個字符串。如果嘗試將錯誤類型的參數傳入 `greet`，TypeScript 編譯器就會報錯。
