# 專業開發者如何與 AI 共舞：打造高效的自動化程式碼審查流程

在這個 AI 協作開發的時代，我們面臨著如何有效地審查 AI 生成的程式碼的挑戰。本文將著重在建立 Coding Standard 的核心思維：「機器能做的交給自動化，人類專注於邏輯與架構。」我們將探討如何讓團隊能高效審查 AI 生成的 Python 與 Terraform 程式碼，並建立四層架構的自動化治理體系。

## 自動化的核心：讓機器做機器該做的事

首先，我們不應在 Pull Request (PR) 中浪費時間討論縮排或變數命名。這些可以由機器自動完成。例如，我們可以利用 Pre-commit hooks，確保 AI 生成的程式碼在進入 Git 之前就已經被「校正」過。

Python 的工具組合推薦使用 Ruff 和 MyPy，Ruff 是目前最推薦的工具，速度極快，集成了 Linter 和 Formatter（可取代 Flake8, Isort, Black）。MyPy 功能強大，可強制執行型別檢查，AI 在有 Type Hints 的環境下生成的程式碼準確度更高。

對於 Terraform，我們可以使用 terraform fmt 進行基本格式化，TFLint 來檢查是否遵循雲端服務的最佳實踐，Checkov 或 tfsec 則可以檢查 AI 是否生成了具備資安風險的設定。
